-- Created a custom user model - models.py

from django.db import models
from django.contrib.auth.models import AbstractUser


class User(AbstractUser):

    ROLE_CHOICES = (
        ('admin', 'Admin'),
        ('support', 'Support Engineer'),
        ('customer', 'Customer'),
    )

    role = models.CharField(
        max_length=20,
        choices=ROLE_CHOICES,
        default='customer'
    )

    phone_number = models.CharField(
        max_length=15,
        blank=True,
        null=True
    )

    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.username

Why This Is Important

Now your user table supports:

roles
extra fields
future scalability

This is real enterprise design.

-- Tell Django to Use Custom User Model

In Django:

ALWAYS create custom user model BEFORE first migrate.

Because once auth tables are created, changing user model becomes difficult.

This is actually an important real-world Django concept.
